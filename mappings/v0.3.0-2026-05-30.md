# Verification Gate Mapping — `v0.3.0-2026-05-30`

**Mapping ID:** `v0.3.0-2026-05-30`
**Status:** Published, immutable.
**ADR:** [ADR-002](../adr/ADR-002-canonical-derived-version-binding.md)
**Effective from:** receipt spec v0.3.0
**Effective until:** superseded by next published mapping; receipts signed against this ID remain verifiable against this document indefinitely.

## Recommendation enum

A receipt's `v_recommendation` is computed from signed primitives `v_verdict`, `v_confidence`, `v_gate_threshold`, `v_adversarial_result` as:

| `v_verdict` | `v_confidence` vs `v_gate_threshold` | `v_adversarial_result` | `v_recommendation` |
|---|---|---|---|
| `supported` | `≥` | `resilient` or `not_checked` | `confident_supported` |
| `supported` | (any) | `vulnerable` | `vulnerable_supported` |
| `supported` | `<` | `resilient` or `not_checked` | `weak_supported` |
| `refuted` | (any) | (any) | `refuted` |
| `unverifiable` | (any) | (any) | `unverifiable` |
| (any error path: timeout, no sources, runtime error) | — | — | `error` |

## Gate mapping

Under this mapping ID, `v_gate` is computed from `v_recommendation` as:

```
gate(rec) = "act"   if rec == "confident_supported"
gate(rec) = "halt"  otherwise
```

## Verifier behavior

A verifier holding a receipt with `v_gate_mapping == "v0.3.0-2026-05-30"` MUST:

1. Fetch this document (or a cached copy) by mapping ID.
2. Recompute `candidate_recommendation` from signed primitives.
3. Assert `candidate_recommendation == v_recommendation`.
4. Apply the gate function above to `v_recommendation` to produce `candidate_gate`.
5. Assert `candidate_gate == v_gate`.

Any mismatch is a malformed receipt; the relying party MUST treat it as `halt`.

## Notes

- This mapping is intentionally narrow: only `confident_supported` produces `act`. All weaker affirmatives, all refutations, all uncertainty, all error paths halt.
- Future mappings (e.g. `v0.4.0-...`) MAY introduce additional `act`-eligible recommendation values (for example, a `qualified_supported` that admits slightly lower confidence under specific source-quality conditions). When that happens, receipts signed under `v0.3.0-2026-05-30` continue to validate against this document; only new receipts will reference the new mapping.
- This file is immutable after first publication. Errata are logged at the top of the file but the table above is frozen.
